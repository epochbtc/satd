# satd 0.2.1

> Released 2026-05-29 · git tag [`v0.2.1`](https://github.com/epochbtc/satd/releases/tag/v0.2.1) · [Changelog](../../CHANGELOG.md)

A packaging-only patch release. **No code changed from 0.2.0** — this release
exists solely to ship the `sat-tui` binary in the release tarballs, which was a
workspace member but had never been built or staged into the archive. Operators
running 0.2.0 from source or container are unaffected; only those installing
from a release tarball need it to get the TUI.

## Highlights

- **`sat-tui` now ships in the release tarballs** alongside `satd` and
  `sat-cli`, with its own SBOM and minisign signature.

## Packaging

- **`sat-tui` is now included in the release tarballs.** The terminal
  dashboard (`sat-tui`) ships in `bin/` of every
  `satd-<version>-<target>.tar.zst` alongside `satd` and `sat-cli`, with a
  CycloneDX SBOM (`sat-tui-v<version>.cdx.json`) and a minisign signature like
  the other artifacts. It was a workspace member but was never built or staged
  into the tarball through 0.2.0, so operators who installed from a release
  archive had no TUI. No code changed from 0.2.0 — this release exists solely to
  package the binary.

## Upgrade notes

Drop-in upgrade from 0.2.0; no behavior, config, or storage changes. Tarball
users gain the `sat-tui` binary.

## About this release

satd is free software under the [MIT License](../../LICENSE).

- **Stability & compatibility** — Tier 1 surfaces (Bitcoin Core-compatible
  JSON-RPC wire shape, CLI flag names/defaults, `bitcoin.conf` keys, on-disk
  layout, Electrum/Esplora surfaces) are governed by
  [`STABILITY_POLICY.md`](../../STABILITY_POLICY.md). satd follows
  [semantic versioning](https://semver.org/spec/v2.0.0.html); while in the
  `0.x` pre-1.0 line, deprecations may be accelerated (see the policy).
- **Security & disclosure** — report vulnerabilities per
  [`SECURITY.md`](../../SECURITY.md). Consensus-affecting bugs are treated as
  P0 with same-day acknowledgement.
- **Verifying this release** — tarballs are signed with minisign, container
  images with cosign (keyless OIDC), and git tags with SSH signatures (no GPG).
  The key matrix and verification steps are in
  [`SECURITY.md`](../../SECURITY.md).
- **Intentional differences from Bitcoin Core** — catalogued in
  [`CORE_DIFFERENCES.md`](../../CORE_DIFFERENCES.md).
- **Operating satd** — observability, the full flag matrix, tuning, and the
  downstream packaging contract live in the
  [Operator Manual](https://epochbtc.github.io/satd/).
